# TinyOS · 从零构建一个 Rust + RISC-V 操作系统

一个面向工程实践的操作系统课程：用 Rust 在 QEMU RISC-V 上从零实现一个可启动、可分时、可分页、可持久化文件、可 shell 交互的类 Unix 内核。12 周、7 个 Phase、20 个 Lab，每一步都有可运行的交付物。

**[📘 在线课程网站](https://example.github.io/tinyos-course/)** · **[🗺️ 课程路线图](./course-roadmap.jsx)**

> 课程受 xv6-riscv（MIT 6.S081）、rCore-Tutorial（清华）、OSTEP（Remzi）的启发，但用 2024-2025 年的工具链（Rust nightly + QEMU 9）与现代工程实践重写。

## Course Overview

| Phase | Topic | 核心概念 | 交付物 |
|:--|:--|:--|:--|
| **0** | Boot & Kernel Entry | no_std Rust · linker script · SBI · panic handler | `tinyos-boot`（QEMU 启动 + println!） |
| **1** | Traps & Syscalls | TrapContext · stvec/sepc/scause · ecall · 首批 syscall | `tinyos-trap`（用户态 write/exit） |
| **2** | Process & Scheduling | TCB · `__switch` 汇编 · 时钟中断 · Round-Robin | `tinyos-proc`（N 个 app 分时） |
| **3** | Concurrency & Locks | SpinLock · SleepLock · Semaphore · Condvar · 死锁 | `tinyos-sync`（哲学家就餐 60s 不死锁） |
| **4** | Virtual Memory | SV39 三级页表 · FrameAllocator · MemorySet · mmap | `tinyos-vm`（每进程独立地址空间） |
| **5** | File System | virtio-blk · easy-fs · BlockCache · VFS | `tinyos-fs`（open/read/write 持久化） |
| **6** | Shell & Pipes | fork/exec/wait · fd table · pipe · 8 个 coreutils | `tinyos v1.0`（交互式 shell，`ls \| wc -l` 正确） |

最终你将得到一个 ~8000 行 Rust + ~200 行汇编的小型 Unix 内核，完整支持多进程、虚拟内存、持久化文件系统与交互 shell。

## Quick Start

### 环境准备
```bash
# macOS
brew install qemu riscv64-elf-gdb

# Ubuntu / Debian
sudo apt install qemu-system-misc gdb-multiarch

# Rust toolchain + RISC-V target
rustup toolchain install nightly-2024-05-02
rustup target add riscv64gc-unknown-none-elf --toolchain nightly-2024-05-02
rustup component add rust-src llvm-tools-preview --toolchain nightly-2024-05-02
```

### 跑通 Phase 0
```bash
cd labs/phase_0_boot
make qemu          # 期望输出：Hello, TinyOS!
make test          # 跑 host-side 单元测试
make grade         # 自动评分（带进度条）
```

### 进入下一 phase
```bash
cd labs/phase_1_trap
cat COURSE.zh-CN.md   # 先读课程文档
# 按文档完成 src/trap/{context.rs,mod.rs,trap.S} 和 src/syscall/*.rs 中的 TODO
make test && make qemu
```

### 启动课程网站（本地预览）
```bash
cd website
npm install
npm run dev        # http://localhost:5173
```

## Tech Stack

| 层 | 选择 | 为什么 |
|---|---|---|
| 内核语言 | Rust nightly + no_std | 内存安全 + zero-cost + 现代语法 |
| 目标架构 | RISC-V 64 (riscv64gc) | 开放 ISA · 教学友好 · QEMU 一流支持 |
| 仿真器 | QEMU virt | 零成本、可脚本化、可 gdb 调试 |
| 固件 | OpenSBI (QEMU 内置) | 提供 M-mode 启动与 console/shutdown 服务 |
| 构建 | cargo + custom linker.ld | 标准 Rust 工作流 |
| 评分 | Python + qemu 管道捕获 | 跨平台、可视进度条 |
| 文档 | Markdown + ASCII 图 | 离线可读、版本友好 |
| 网站 | Vite + React + TypeScript + HashRouter | 快速、可部署到 GitHub Pages |

## Course Design Philosophy

1. **从裸机跑起来，再谈抽象** — 每一层代码都能立刻 `make qemu` 看到效果。
2. **每一层独立可测** — 下层不感知上层存在；调试时能精准回到某个 phase 隔离。
3. **scause / sepc 是调试的罗盘** — 养成第一反应读寄存器的习惯，gdb + QEMU 是必备能力。
4. **先正确，再并发，最后性能** — 跳过任何一步都会让后续调试成本指数级上升。
5. **用户态可见才是真的完成** — 内核特性必须通过 syscall 被用户程序验证。
